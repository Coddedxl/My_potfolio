Road accident analysis

I turn raw data into clear, actionable insights using data tools
like Power bi, Excel, SQL and python

This dashboard shows road accident data in a very simple way. In total, there are 417,883 casualties. Most of them are slight injuries (about 84%), while serious cases are 14% and fatal cases are very small (1.7%). Cars are involved in most accidents (almost 80%), and many accidents happen on single carriage roads. Also, most crashes happen on dry roads, not wet or snowy ones. When we look at location, urban areas have more accidents than rural areas, and most accidents happen during the day time.

The main problem with this data was that it was not clean or well arranged. Some values were missing, like “(blank)” in road types. Some categories were not consistent, like “Urban” and “urban” meaning the same thing but written differently. There were also small errors in numbers and labels that could confuse anyone reading the dashboard. If we used the data like that, the results would not be correct and stakeholders could make wrong decisions.

To solve this, I did simple data cleaning. I removed or fixed missing values, replaced blanks with proper labels, and made sure all categories were consistent (for example, keeping “Urban” and “Rural” in the same format). I also checked the numbers to make sure totals match and corrected any duplicates. After that, I organized the data into clear groups like road type, vehicle type, and road condition so it becomes easy to understand.

Now the data is clean, clear, and reliable. Stakeholders can quickly see where problems are coming from, like more accidents in urban areas, on dry roads, and mostly involving cars. This helps them make better decisions, like improving road safety in busy cities or focusing on driver awareness. The cleaned data makes the dashboard simple to read and easy to trust.
